# Контент-ревизия «Суровая Стройка» — Design Document

**Дата:** 2026-05-30
**Статус:** На ревью
**Контекст:** ревизия контента и разметки существующего сайта (`DESIGN.md`,
`2026-05-28-content-and-architecture-revisions-design.md`). Исходник пожеланий — `контент.txt`.

## Обзор

Наполняем сайт реальным контентом строительной компании, перестраиваем разделы
«Услуги» и «Цены», переименовываем «Портфолио», добавляем тёмную тему и смягчаем
светлую. Весь смысловой контент (услуги, цены-примеры, статьи) редактируется через
**уже существующую админку** (правки «на лету»), название/слоган/контакты — в
`app.config.ts`. Картинки пока не генерим: заводим единую папку, фикс-размеры и файл
с промптами на будущее.

**Принцип на будущее:** дальнейшие правки контента не должны требовать кода —
поэтому всё динамическое идёт через админку/БД, а в коде остаются короткие
комментарии-подсказки (где палитра, где слоты картинок, как работают теги).

## 2. Блок «Как работаем» (главная)

Заменяет секцию «Прозрачные цены» (`SectionPricesTeaser` шапку) **и** фейковую
статистику (`SectionAbout`: 10+ лет, 200+ объектов, 5 лет гарантии — **удалить**).

Четыре честных блока (заголовок + текст):

1. **Примерная стоимость сразу** — по фото и описанию подскажем порядок цены ещё до выезда.
2. **Всё обсудим заранее** — согласуем объём и стоимость до начала, по ходу работ честно корректируем.
3. **Полный цикл работ** — строительство, ремонт, замена фундамента, передвижка построек.
4. **Материалы — наша забота** — под ключ: закупим и привезём по своей цене.

Реализация: статический массив в компоненте секции (это «о компании», меняется редко;
если позже понадобится — вынесем). Лайн-иконки в цвет акцента.

## 3. Информационная архитектура

Навигация (`AppHeader.vue`): **Главная · Услуги · Из практики · Примеры работ и цен · Контакты**

| Раздел | Маршрут | Источник | Изменение |
|--------|---------|----------|-----------|
| Услуги | `/services`, `/services/[slug]` | БД (новая сущность) | **новый раздел с детальными страницами** |
| Из практики | `/portfolio`, `/portfolio/[slug]` | БД (Articles) | переименование «Портфолио» → «Из практики» (только лейблы; маршрут можно оставить `/portfolio`) |
| Примеры работ и цен | `/prices` | БД (ServicePrice → примеры) | переработка модели и разметки |
| Контакты | `/contact` | app.config | правка текста (Тула, без часов) |

**Услуги:** на главной — сетка из 5 карточек (иконка + название + короткое описание),
клик → детальная страница `/services/[slug]` с богатым описанием и **привязанными по
тегу статьями** из «Из практики».

**Связь по тегу:** услуга и пример цены хранят строковый `Tag`; детальная страница
тянет статьи через `GET /api/articles?tag=<tag>`.

## 4. Изменения данных (бэкенд + админка)

> **Важно:** в БД есть боевые данные. Все миграции — **с сохранением данных**.
> Сиды — **идемпотентные, только если таблица пуста** (новая таблица услуг засидится;
> боевые цены/статьи не трогаем и не дублируем).

### 4.1. Новая сущность `Service`

```
Service { Id, Title, Slug, ShortDescription, IconName, Content (HTML), Tag, SortOrder, IsPublished }
```

- `IconName` — имя лайн-иконки (Lucide), `Content` — санитайзенный HTML (как у Article).
- Слой как у статей: `IService Service` / реализация, `ServicesController` (публичный:
  список + по slug), `Admin/AdminServicesController` (CRUD).
- Эндпоинты:
  ```
  GET /api/services            — список (IsPublished)
  GET /api/services/{slug}     — детальная
  GET /api/admin/services      — все (admin)
  POST/PUT/DELETE /api/admin/services
  ```
  ⚠️ Текущий `ServicesController` отдаёт **цены** (`/api/services/prices`). Развести:
  цены переносим под `/api/prices`, а `/api/services` освобождаем под услуги. (Сверить
  и обновить фронтовые вызовы в `useApi`/`useAdminApi`.)
- Админка: страница списка + форма услуги (как у статей, с WYSIWYG для `Content`).
- Детальная страница `pages/services/[slug].vue` + сетка на главной и `/services`.

### 4.2. `Article` — теги

- Добавить поле `Tags` (строка, значения через запятую/пробел) — аддитивная миграция.
- `IArticleService`/`ArticleService`: поддержать фильтр `?tag=` (на уровне БД).
- Публичный `GET /api/articles?tag=foundation&page=1&pageSize=12`.
- Админ-редактор статьи: поле ввода тегов.

### 4.3. `ServicePrice` → модель «пример работы»

Старое: `Category, Name, Description, PriceFrom, PriceTo, Unit, SortOrder`.
Новое:

```
PriceExample { Id, Title, PhotoPath, Description, Price (int), Duration (string), ArticleSlug?, Tag?, SortOrder }
```

- **Миграция с переносом:** `Title ← Name`, `Description ← Description`,
  `Price ← PriceFrom`, `SortOrder ← SortOrder`; `Category/Unit/PriceTo` — удаляем
  (значения не переносим, они больше не нужны); `PhotoPath/Duration/ArticleSlug/Tag` —
  новые, nullable. Сущность можно оставить классом `ServicePrice` (переименование
  опционально) — главное переработать поля и DTO.
- `Duration` — свободная строка («2 дня», «1 неделя»). `Price` — целое (рубли),
  на фронте форматируем («340 000 ₽» или «от 340 тыс»). Без единиц/метрик.
- `ArticleSlug` — опциональная ссылка на статью с подробностями.
- Правки: `ServicePriceService`, DTO (`ServicePriceDto`/Create/Update),
  `lib/prices.ts` (группировка по Category больше не актуальна — убрать),
  `PriceTable.vue`/`SectionPricesTeaser.vue` → карточки примеров,
  страница `pages/admin/prices.vue` (форма под новые поля + загрузка фото).

### 4.4. Сиды (идемпотентные)

- **Услуги (новая таблица, пуста)** — засидить 5 услуг:
  1. Строительство — дома, беседки, сараи и подобное (по нашему или вашему проекту)
  2. Ремонт и пристройки — веранды, пристройки, расширение, фасады
  3. Замена фундамента — *(без уточнений)*
  4. Передвижка построек — аккуратно: бытовки и небольшие строения
  5. Под ключ — закупим материалы по своей цене и привезём
- **Примеры цен** — засидить **только если таблица пуста** (на проде есть боевые —
  не дублируем):
  1. Замена фундамента дачного дома 4×5 с верандой: с бетонных блоков на винтовые сваи
     и швеллеры. 340 000 ₽, 2 дня. (ссылка на статью про сваи)
  2. Утеплённая каркасная пристройка 3×7, сэндвич-панели, 2 окна. 750 000 ₽, 1 неделя.
- **Статьи** — засидить **только если пусто**: 1–2 статьи на основе истории про замену
  фундамента на сваи/швеллеры (с тегом `foundation`), чтобы продемонстрировать связь
  услуга/цена → статья.

## 5. Тема: «Холодный графит» (светлая + тёмная)

### 5.1. Механика

- Цвета — в **CSS-переменных**: `:root { --color-* }` для светлой, `.dark { ... }`
  для тёмной (в `assets/css/main.css`). Tailwind-токены завязываем на переменные
  (`tailwind.config.ts`): `surface`, `surface-2`, `ink`, `muted`, `base` (border),
  `brand`. В компонентах хардкод `bg-white`, `from-ink`, `text-ink` заменяем на
  семантические токены.
- Переключатель темы в `AppHeader.vue`. По умолчанию — системная (`prefers-color-scheme`),
  выбор сохраняется в `localStorage`. Установка класса `.dark` на `<html>` **до**
  рендера (inline-скрипт в `app.head`/плагин), чтобы не было вспышки светлой темы.
- Учесть `prose` (Tailwind Typography) в тёмной теме — `dark:prose-invert` для контента
  статей/услуг.

### 5.2. Палитра

| Токен | Светлая | Тёмная |
|-------|---------|--------|
| `surface` (фон) | `#f8fafc` | `#0f172a` |
| `surface-2` (карточки) | `#ffffff` | `#1e293b` |
| `base` (границы) | `#e2e8f0` | `#334155` |
| `ink` (текст) | `#1f2937` | `#f1f5f9` |
| `muted` | `#64748b` | `#94a3b8` |
| `brand` (акцент) | `#d97706` | `#f59e0b` |

Геро — тёмный градиент в обеих темах (в светлой `#1f2937→#374151`, в тёмной
`#020617→#1e293b`).

## 6. Иконки и картинки

- **Иконки:** лайн-иконки Lucide (через `nuxt-lucide-icons` или инлайн-SVG) в цвет
  акцента — для карточек услуг и блоков «Как работаем». Эмодзи убрать.
- **Картинки (пока без генерации):**
  - Папка `frontend/public/images/` — единое место. Плейсхолдер `placeholder.svg`.
  - Все слоты — фиксированный `aspect-ratio` + `object-fit: cover`, чтобы замена
    файла не ломала вёрстку и не требовала обработки. Рекомендованные размеры — в
    `docs/image-prompts.md`.
  - `docs/image-prompts.md` — готовые промпты для нейросети по каждому слоту (геро-фон,
    5 услуг, плейсхолдер) + рекомендованные размеры/пропорции. Сгенеришь позже — просто
    положишь файлы с нужными именами.
- Фото из админки (статьи, примеры цен) — по-прежнему `/uploads` (динамика), это
  отдельный от `public/images` поток.

## 7. Объём работ (границы)

**Входит:** тексты/бренд, новая сущность Service + детальные страницы, теги статей,
переработка модели цен, тема (светлая+тёмная) + переключатель, лайн-иконки, папка
картинок + промпты, идемпотентные сиды, обновление тестов под изменённые модели/эндпоинты.

**Не входит:** генерация картинок (промпты — да, файлы — позже), редизайн админки сверх
необходимого для новых полей, конструктор сайта, изменение деплоя/инфраструктуры.
